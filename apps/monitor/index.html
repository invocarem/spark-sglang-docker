SGLang Stack Dashboard

About this dashboard

Launch starts SGLang from scripts/*.sh, detects launch_server, and resolves the served model id. Set the model id for Chat/Benchmark; other tabs: Logs, Docker tools, metrics, benchmark, chat.

Start SGLang

Pick a container and a script from ./scripts, then run or check status. While launch_server is running, starting again is blocked. Script output while the model loads is on the Logs tab (launch script log).

How it works

Host repo ./scripts is mounted at /workspace/scripts. The API runs docker exec -d … with the script’s stdout/stderr appended to /workspace/.monitor/sglang-launch.log (that output does not appear in docker logs, which is only the container’s main process). Open the Logs tab to tail that file; Check launch server and SGLang metrics show readiness.

Chat & Benchmark model id

Must match --served-model-name from your script. Saved in this browser; Check launch server syncs it to the running server’s model id when they differ.

Container logs

Two different sources: Docker logs is the container’s main process only (same as docker logs on the host). Launch script log is /workspace/.monitor/sglang-launch.log (output from scripts started on the Launch tab — it does not appear in docker logs).
